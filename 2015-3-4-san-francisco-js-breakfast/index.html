Immutable deploys

The case for immutability

Mutable infrastructure

  • Anyone with SSH can change production code (happens)
  • Deploys that overwrite old files might not make clean update
  • Risk of css/js build failing leaving broken css or weird JS errors in production (happened all the time at last startup)
  • Rolling back code isn't clean or fast without complicated backup scheme
  • Testing is hard

Immutable infrastructure (with Docker)

  • Can't change system once its built. If you need to change it, deploy a new system.
  • Easy to test image once built to ensure it'll work as expected
  • Rolling back is trivial
  • Crash only architecture. Stuff is going to break, have bugs, need updates. Make it really easy to throw away the old and bring in the new.
